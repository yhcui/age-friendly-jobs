# 贡献指南 | Contributing Guide

感谢你愿意为这个项目贡献力量！🙏

## 快速开始

### 1. Fork 项目

点击 GitHub 页面右上角的 Fork 按钮

### 2. 克隆到本地

```bash
git clone https://github.com/你的用户名/age-friendly-jobs.git
cd age-friendly-jobs
```

### 3. 创建分支

```bash
git checkout -b add-company-xxx
```

### 4. 添加企业信息

在 `companies/friendly/` 对应城市文件中添加：

```markdown
### 公司名称

- **行业**：互联网/金融科技/智能制造/...
- **规模**：500-1000 人
- **友好岗位**：后端开发/产品经理/数据分析
- **年龄范围**：35-45 岁均有在职
- **信息来源**：公司官网招聘页/员工背书
- **验证状态**：待验证/已验证（1 人背书）/已验证（多人背书）
- **备注**：面试不问年龄，更看重项目经验
- **更新时间**：2026-01-01
```

### 5. 提交 PR

```bash
git add .
git commit -m "添加：XXX 公司（北京）"
git push origin add-company-xxx
```

然后在 GitHub 上创建 Pull Request

## 验证标准

### 企业收录标准

| 级别 | 要求 | 标识 |
|------|------|------|
| 待验证 | 公开信息显示年龄友好 | ⏳ |
| 已验证 | 1 名员工背书 | ✅ |
| 已验证 | 3 名 + 员工背书 | ✅✅✅ |

### 员工背书格式

通过 Issue 或 PR Comment 形式：

```
【背书验证】
公司：XXX 公司
在职状态：在职/离职（X 年 -X 年）
年龄：40 岁
岗位：高级开发工程师
验证内容：团队 35 岁 + 同事占比约 40%，无年龄歧视
联系方式（可选）：email/微信
```

## 注意事项

### ✅ 鼓励的行为

- 提供真实、可验证的信息
- 附上信息来源链接
- 客观描述，避免情绪化
- 及时更新过期信息

### ❌ 避免的行为

- 恶意抹黑或虚假宣传
- 泄露公司内部机密
- 人身攻击或情绪发泄
- 商业用途或广告推广

## 常见问题

### Q: 我只是听说某公司友好，可以提交吗？

A: 可以，但请注明"信息来源：网络传闻/朋友推荐"，验证状态标记为"待验证"

### Q: 我在某公司工作，想背书但怕被识别怎么办？

A: 可以匿名背书，通过 Issue 私密提交或私信维护者

### Q: 发现信息不准确怎么办？

A: 直接提 Issue 或 PR 修正，附上正确信息
